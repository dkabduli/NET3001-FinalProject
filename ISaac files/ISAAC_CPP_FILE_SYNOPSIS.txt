src/Buzzer.cpp
What it does:
- Implements buzzer behavior.
How it worls:
- Buzzer_init() drives BUZ low at startup.
- Buzzer_beep_once() sets BUZ high, delays, then sets BUZ low.
Why it matters:
- Gives audible warning on violation.

src/LCD.cpp
What it does:
- Implements HD44780 LCD driver in 4-bit mode.
How it worls:
- lcd_nibble_write() sends one nibble and EN pulse.
- lcd_byte_write() sends two nibbles for one byte.
- LCD_init() runs startup commands; LCD_line1()/LCD_line2() write padded text.
Why it matters:
- Main user-facing text display output.

src/SevenSeg.cpp
What it does:
- Implements shift-register-driven 7-segment display.
How it worls:
- sr_send() shifts 8 bits using SR_SER and pulse_clk().
- latch() copies shifted bits to outputs.
- SevenSeg_show_digit() maps digit with table then calls sr_send() + latch().
Why it matters:
- Displays countdown/numeric info using fewer pins.

src/Ultrasonic.cpp
What it does:
- Implements ultrasonic distance measurement.
How it worls:
- Ultrasound_read_cm() sends trigger pulse on SONIC_TRIG.
- It measures echo high time from SONIC_ECHO_PIN with timeout checks.
- It returns distance using us/58 conversion.
Why it matters:
- Detects object movement for violation checks.

src/TrafficLight.cpp
What it does:
- Implements traffic light state machine.
How it worls:
- Uses TrafficLight_init() and TrafficLight_step() as public flow.
- Uses helpers next_phase(), tick_one_second(), poll_red_camera(), handle_violation(), lcd_normal(), and LED helpers.
- Calls LCD_line1/2(), SevenSeg_show_digit(), Buzzer_beep_once(), Ultrasound_read_cm(), USART0_print(), and timer/button consume functions.
Why it matters:
- Core project logic coordinating all modules.
