src/GPIO.cpp
What it does:
- Configures directions and safe defaults for all pins.
How it worls:
- Uses helper functions out1() and in_pullup() inside GPIO_init().
- GPIO_init() sets LED pins, buzzer pin, ultrasonic pins, shift-reg pins, button pullups, and LCD pins.
Why it matters:
- Prevents bad startup pin states and wiring issues.

src/PCINT_buttons.cpp
What it does:
- Handles pause/reset pin-change interrupts and debounce.
How it worls:
- ISR(PCINT0_vect) sets s_pause_edge and ISR(PCINT1_vect) sets s_reset_edge.
- PCINT_consume_pause_click() and PCINT_consume_reset_click() call take_pause()/take_reset(), then stable_low_pause()/stable_low_reset() with debounce delay.
Why it matters:
- Prevents false or repeated button triggers.

src/main.cpp
What it does:
- Runs startup setup and main loop calls.
How it worls:
- setup() calls GPIO_init(), USART0_init_9600(), LCD_init(), SevenSeg_init(), Buzzer_init(), Ultrasound_init(), PCINT_buttons_init(), Timer1_init_1Hz_tick(), TrafficLight_init(), then sei().
- loop() repeatedly calls TrafficLight_step().
Why it matters:
- Entry point that ties all modules together.

src/Timer1Tick.cpp
What it does:
- Generates a 1-second periodic tick.
How it worls:
- Timer1_init_1Hz_tick() sets CTC mode and OCR1A.
- ISR(TIMER1_COMPA_vect) sets s_pending_1s, and Timer1_consume_1s_tick() returns/clears that flag atomically.
Why it matters:
- Enables non-blocking countdown timing.

src/TrafficLight.cpp
What it does:
- Implements traffic state machine and violation handling.
How it worls:
- Main flow is TrafficLight_init() + TrafficLight_step().
- Helper functions next_phase(), tick_one_second(), poll_red_camera(), handle_violation(), lcd_normal(), leds_only_green()/amber()/red() control behavior.
- Uses PCINT_consume_pause_click(), PCINT_consume_reset_click(), Timer1_consume_1s_tick(), Ultrasound_read_cm(), LCD_line1/2(), SevenSeg_show_digit(), Buzzer_beep_once(), USART0_print(), USART0_print_u16().
Why it matters:
- Core behavior of the full project.

src/USART0.cpp
What it does:
- Implements UART register setup and serial printing.
How it worls:
- USART0_init_9600() configures UBRR0H/UBRR0L/UCSR0A/UCSR0B/UCSR0C.
- USART0_tx_byte() waits on UDRE0 then writes UDR0.
- USART0_print() sends strings and USART0_print_u16() calls print_digits() for decimal conversion.
Why it matters:
- Gives runtime visibility for debugging and logs.
