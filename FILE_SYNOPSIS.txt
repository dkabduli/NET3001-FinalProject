include/LCD.h
What it does:
- Declares the LCD public API.
How it works:
- Exposes LCD_init(), LCD_clear(), LCD_line1(), LCD_line2().
Why it matters:
- Other modules can use LCD without touching low-level LCD logic.
Specfic thign it does linked to this function:
- LCD_line1() / LCD_line2() are implemented in src/LCD.cpp.

include/USART0.h
What it does:
- Declares serial (UART) public API.
How it works:
- Exposes USART0_init_9600(), USART0_print(), USART0_print_u16().
Why it matters:
- Gives one consistent debug/logging interface.
Specfic thign it does linked to this function:
- USART0_init_9600() is implemented in src/USART0.cpp.

include/SevenSeg.h
What it does:
- Declares 7-segment display API (ShiftReg-backed).
How it works:
- Exposes SevenSeg_init() and SevenSeg_show_digit().
Why it matters:
- Lets app code show numbers without shift-register protocol details.
Specfic thign it does linked to this function:
- SevenSeg_show_digit() is implemented in src/SevenSeg.cpp.

include/Buzzer.h
What it does:
- Declares buzzer control API.
How it works:
- Exposes Buzzer_init() and Buzzer_beep_once().
Why it matters:
- Provides simple audible alert trigger.
Specfic thign it does linked to this function:
- Buzzer_beep_once() is implemented in src/Buzzer.cpp.

include/GPIO.h
What it does:
- Declares GPIO bootstrap API.
How it works:
- Exposes single setup function GPIO_init().
Why it matters:
- Centralizes pin init so all modules start with correct pin states.
Specfic thign it does linked to this function:
- GPIO_init() is implemented in src/GPIO.cpp.

include/PCINT_buttons.h
What it does:
- Declares pause/reset interrupt+debounce API.
How it works:
- Exposes PCINT_buttons_init(), PCINT_consume_pause_click(), PCINT_consume_reset_click().
Why it matters:
- Converts noisy button edges into clean click events.
Specfic thign it does linked to this function:
- PCINT_consume_pause_click() is implemented in src/PCINT_buttons.cpp.

include/Timer1Tick.h
What it does:
- Declares 1-second timer tick API.
How it works:
- Exposes Timer1_init_1Hz_tick() and Timer1_consume_1s_tick().
Why it matters:
- Keeps timing logic separate from traffic logic.
Specfic thign it does linked to this function:
- Timer1_consume_1s_tick() is implemented in src/Timer1Tick.cpp.

include/TrafficLight.h
What it does:
- Declares top-level traffic controller API.
How it works:
- Exposes TrafficLight_init() and TrafficLight_step().
Why it matters:
- Clean entry points for startup and loop execution.
Specfic thign it does linked to this function:
- TrafficLight_step() is implemented in src/TrafficLight.cpp.

include/Ultrasonic.h
What it does:
- Declares ultrasonic distance API.
How it works:
- Exposes Ultrasound_init() and Ultrasound_read_cm().
Why it matters:
- Supplies distance input for red-light violation detection.
Specfic thign it does linked to this function:
- Ultrasound_read_cm() is implemented in src/Ultrasonic.cpp.

include/PinMap.h
What it does:
- Defines all pin mappings/macros.
How it works:
- Maps logical names (LED/LCD/SR/button/etc.) to DDR/PORT/PIN bits.
Why it matters:
- Wiring changes happen in one place only.
Specfic thign it does linked to this function:
- Used by GPIO_init() in src/GPIO.cpp and by most drivers.

src/main.cpp
What it does:
- System startup and runtime loop.
How it works:
- setup() initializes modules; loop() repeatedly calls TrafficLight_step().
Why it matters:
- Entry point that wires all modules together.
Specfic thign it does linked to this function:
- Calls TrafficLight_step() defined in src/TrafficLight.cpp.

src/GPIO.cpp
What it does:
- Configures pin directions and safe defaults.
How it works:
- Uses local helpers out1() and in_pullup() inside GPIO_init().
Why it matters:
- Prevents undefined/floating pin behavior at boot.
Specfic thign it does linked to this function:
- GPIO_init() is called from setup() in src/main.cpp.

src/USART0.cpp
What it does:
- Implements UART init and serial printing.
How it works:
- USART0_init_9600() sets UART registers; USART0_tx_byte() waits for UDRE then writes UDR0.
Why it matters:
- Gives reliable debug/event logs.
Specfic thign it does linked to this function:
- USART0_print_u16() is called in handle_violation() in src/TrafficLight.cpp.

src/LCD.cpp
What it does:
- Implements 16x2 LCD driver in 4-bit mode.
How it works:
- lcd_nibble_write() + lcd_byte_write() + LCD_init() startup sequence + line writers.
Why it matters:
- Main on-device text feedback.
Specfic thign it does linked to this function:
- LCD_line1() / LCD_line2() are called in lcd_normal() and handle_violation() in src/TrafficLight.cpp.

src/SevenSeg.cpp
What it does:
- Implements shift-register-driven 7-segment output.
How it works:
- sr_send() shifts 8 bits; latch() updates outputs; SevenSeg_show_digit() maps 0-9.
Why it matters:
- Displays numeric countdown with fewer MCU pins.
Specfic thign it does linked to this function:
- SevenSeg_show_digit() is called in multiple places in src/TrafficLight.cpp.

src/Buzzer.cpp
What it does:
- Implements buzzer output behavior.
How it works:
- Buzzer_beep_once() sets buzzer pin high briefly then low.
Why it matters:
- Audible violation alert.
Specfic thign it does linked to this function:
- Buzzer_beep_once() is called in handle_violation() in src/TrafficLight.cpp.

src/Ultrasonic.cpp
What it does:
- Reads distance from ultrasonic sensor.
How it works:
- Sends trigger pulse, times echo pulse width, converts to cm.
Why it matters:
- Detects crossing movement during red phase.
Specfic thign it does linked to this function:
- Ultrasound_read_cm() is called in poll_red_camera() in src/TrafficLight.cpp.

src/PCINT_buttons.cpp
What it does:
- Handles button interrupts and debounce.
How it works:
- ISR sets edge flags, consume functions debounce and wait for release.
Why it matters:
- Prevents false/double button triggers.
Specfic thign it does linked to this function:
- PCINT_consume_pause_click() / PCINT_consume_reset_click() are called in TrafficLight_step() in src/TrafficLight.cpp.

src/Timer1Tick.cpp
What it does:
- Generates 1-second tick events.
How it works:
- Timer1 compare ISR sets s_pending_1s; consume function atomically returns/clears it.
Why it matters:
- Non-blocking timing for phase countdown.
Specfic thign it does linked to this function:
- Timer1_consume_1s_tick() is called in TrafficLight_step() in src/TrafficLight.cpp.

src/TrafficLight.cpp
What it does:
- Core traffic state machine and violation detection.
How it works:
- Manages phases, countdown, button state, ultrasonic checks, and output modules.
Why it matters:
- This is the main control logic of the project.
Specfic thign it does linked to this function:
- TrafficLight_step() is called from loop() in src/main.cpp.
